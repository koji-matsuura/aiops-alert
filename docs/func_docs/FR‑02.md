# FR‑02 Bottleneck Investigation

## 概要
Agent が CloudWatch Metrics（CPU, Memory, RDS の `free_storage_space_bytes` 等）を取得し、
最もボトルネックが起こりそうなリソースを特定します。
検出したボトルネックはチャット・ダッシュボードに推奨アクションとして提示します。

## 入力 | 主要パラメータ
| パラメータ | 型 | 説明 |
|-----------|----|------|
| `TimeRangeSeconds` | integer | 監視期間（秒） - デフォルト 900（15 min） |
| `Thresholds` | map | 各メトリクスに対するしきい値例 `{"CPU": 90, "FreeStorageSpace": 100000}` |
| `ResourceARNs` | array | 取得対象のリソース ARN リスト |
| `MaxResults` | integer | 取得件数上限（デフォルト 100） |

## 処理フロー
1. EventBridge（cron: `0/15 * * * ? *`）で Lambda を起動。  
2. `cloudwatch:ListMetrics` で指定リソースの `CPUUtilization`, `FreeStorageSpace` 等を取得。  
3. しきい値超過を検知、最も高いスコアを持つリソースを **ボトルネック** と判断。  
4. 推奨アクション（スケールアウト・ストレージ追加等）を OpenAPI で呼び出し、SNS で通知。  
5. 結果は CloudWatch Logs (`/aws/lambda/aiops-bottleneck`) に保存、Metric `AiopsBottleneckDetected` を更新。

## SNS 通知設定
| SNS Topic | 内容 | 受信者 |
|-----------|------|--------|
| `arn:aws:sns:ap-northeast-1:123456789012:BottleneckReport` | BOTTLENECK REPORT | チャット、メール |

## IAM ポリシー（Lambda 実行ロール）
```json
{
  "Effect": "Allow",
  "Action": [
    "cloudwatch:ListMetrics",
    "cloudwatch:GetMetricStatistics",
    "cloudwatch:PutMetricData",
    "sns:Publish",
    "logs:CreateLogGroup",
    "logs:CreateLogStream",
    "logs:PutLogEvents",
    "lambda:InvokeFunction"
  ],
  "Resource": "*"
}
```

## デプロイ手順（CDK なし）
1. Lambda ハンドラを `/opt/lambda/bottleneck/` に配置（Python/Node.js 例）。  
2. IAM ロールを作成し、上記ポリシーを添付。  
3. `aws lambda create-function` で関数を作成。  
4. `aws events put-rule` で `cron(0/15 * * * ? *)` を作成し、Lambda をターゲットに設定。  
5. SNS トピックを作成し、必要なサブスクライバーを追加。
