# FR‑01 Log Investigation

## 概要
CloudWatch Logs に蓄積される **すべてのログ・トレース** を対象に、\n1〜2 分間隔のログを集約・解析し、\n**運用インシデント** や **セキュリティ異常** を自動検知します。\n検出結果は SNS (`LogInvestigationReport`) へ通知し、関係者や自動修復タスクに転送します。

## 入力 | 主要パラメータ

| パラメータ | 型   | 説明 |
|-----------|------|------|
| `LogGroupNamePrefix` | string | 監視対象ロググループ名のプレフィックス（例：`/aws/lambda/`）。 |
| `TimeRangeSeconds`   | integer | 取得開始時刻からの秒数（デフォルト 900 s = 15 min）。 |
| `FilterPattern`      | string | CloudWatch Logs Insight フィルター。例：`"?ERROR *"` |
| `MaxResults`         | integer | 取得件数上限（デフォルト 1000）。 |

## 処理フロー

1. **EventBridge スケジューラ**\n   毎分または毎日（例：`cron(0/15 * * * ? *)`）に Lambda を起動。\n
2. **ログ収集**\n   `FilterLogEvents` API を呼び出し、`FilterPattern` で一致したイベントを抽出。\n   - 取得期間は `StartTime` = `Now - TimeRangeSeconds`。\n   - `LogGroupNamePrefix` にマッチするすべてのロンググループを走査。\n
3. **インシデント検出**\n   取得したログを解析し、以下の 3 つのルールで異常を特定。\n   - **エラー・例外**：`ERROR`, `EXCEPTION` 等のキーワード。\n   - **高頻度**：一定時間内に同一メッセージが `N` 回以上出現。\n   - **セキュリティ異常**：`FailedLogin`, `Unauthorized` など。\n
4. **レポート生成**\n   イベントごとに以下の構造を JSON でまとめ、SNS に publish。\n   ```json\n   {\n     "type": "logInvestigation",\n     "runAt": "2026-06-10T10:00:00Z",\n     "logGroup": "/aws/lambda/my-function",\n     "alerts": [\n       {\n         "level": "ERROR",\n         "message": "NullPointerException in Handler",\n         "timestamp": "2026-06-10T09:58:42Z",\n         "count": 1\n       }\n     ]\n   }\n   ```\n
5. **ログバックアップ**\n   解析済みログは **S3 バケット** (`aiops-logs/processed/`) にバックアップ。\n   アーカイブは 90 日保管。\n
6. **統計送信**\n   監査向けに `CloudWatch Metric`（例：`LogErrors`）を更新し、ダッシュボードで可視化。\n
## SNS 通知設定

| SNS Topic | 送信メッセージ | 受信者 |
|-----------|----------------|---------|
| `arn:aws:sns:ap-northeast-1:123456789012:LogInvestigationReport` | FR‑01 レポート | メール、Slack、PagerDuty など |

## IAM ポリシー（Lambda 実行ロール）

```json
{\n  "Effect": "Allow",\n  "Action": [\n    "logs:DescribeLogGroups",\n    "logs:FilterLogEvents",\n    "logs:DescribeLogStreams",\n    "cloudwatch:PutMetricData",\n    "sns:Publish",\n    "s3:PutObject"\n  ],\n  "Resource": "*"\n}
```\n
## デプロイ手順（CDK なし）

1. **Lambda ハンドラ** を `/opt/lambda/loginvestigator/` に配置（Python/Node.js 例）。\n2. **IAM ロール** `CreateRole.sh` で作成し、上記ポリシーを添付。\n3. `aws lambda create-function` で関数を作成。\n4. **EventBridge ルール** `aws events put-rule` で `cron(0/15 * * * ? *)` を作成し、Lambda をターゲットに設定。\n5. **SNS トピック** を作成し、必要なサブスクライバーを追加。\n6. **S3 バケット** `aiops-logs` を作成し、Lambda に書き込み権限を付与。\n
