# FR‑06 高負荷クエリ調査

## 概要
RDS Performance Insights を利用し、**CPU, I/O, ネットワーク** の 90％超のリソース使用で発生したクエリを検出。結果は SNS (`HighLoadQueryReport`) へ通知。

## 入力 | パラメータ
| パラメータ | 型 | 説明 |
|-----------|----|--------|
| `Duration` | integer | 監視期間（秒）。デフォルト 86400（24h）。 |
| `ThresholdPercent` | float | 90.0 などのパーセンテージ。 |
| `Metrics` | list | 監視対象メトリクス。例：`["CPUUtilization", "DiskIOPS", "NetworkTransmitThroughput"]` |

## 処理フロー
1. EventBridge で毎週日曜 00:00 UTC に Lambda を起動。
2. `GetResourceMetrics` を呼び出し、指定したメトリクスを取得。
3. 取得したメトリクスが `ThresholdPercent` を超えたエントリを **High‑Load** と判別。
4. SQL と統計情報（平均 CPU, I/O, ネットワーク）を集計し、JSON で SNS に publish。
5. 実行ログは `CloudWatch Logs /aws/lambda/aiops-highload-query-detector` に残す。

## 出力 | レポートJSON
```json
{
  "type": "highLoadQueries",
  "runAt": "2026-06-10T00:00:00Z",
  "queries": [
    {
      "queryText": "CALL update_inventory()",
      "avgCpuPercent": 92.3,
      "avgDiskIoPercent": 88.1,
      "timeRange": "22:00-23:00"
    }
  ]
}
```

## SNS 通知設定
- **トピック**：`arn:aws:sns:ap-northeast-1:123456789012:HighLoadQueryReport`
- **サブスクライバー**：メール, SQS, Slack webhook に設定可能。

## デプロイ手順（CDK なし）
1. Lambda ハンドラを `/opt/lambda/highload/` ディレクトリへ配置。
2. `create-iam-role.sh` で `PerformanceInsightsRole` を作成（上記 IAM ポリシーを添付）。
3. `aws lambda create-function` で Lambda を作成。
4. `aws events put-rule` でバッチスケジュール作成。
5. `aws events put-targets` で Lambda をターゲット設定。
6. SNS トピック作成し、`aws sns subscribe` で購読追加。
