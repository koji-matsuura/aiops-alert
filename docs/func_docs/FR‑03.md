# FR‑03 DB Snapshot Creation

## 概要
Agent が指定 RDS インスタンスのスナップショットを作成し、インシデント ID でタグ付けします。
手動トリガ（Agent のプロンプト）または **FR‑02** の自動応答から呼び出します。

## 入力 | 主要パラメータ
| パラメータ | 型 | 説明 |
|-----------|----|------|
| `DBInstanceIdentifier` | string | ターゲット RDS インスタンス名 |
| `SnapshotName` | string | （任意）スナップショット名。未指定時は自動生成 (`snap-YYYYMMDD-HHMM-${DBInstance}`)。 |
| `Tags` | map | インシデント ID などの任意タグ。 |

## 処理フロー
1. Lambda が `rds:CreateDBSnapshot` を呼び出し、スナップショット作成を開始。  
2. スナップショットが完成したら `SnapshotArn` を取得。  
3. SNS (`SnapshotReport`) に ARN とメタデータを publish。  
4. 同時に CloudWatch Log (`/aws/lambda/aiops-snapshot`) へログを残す。

## SNS 通知設定
| SNS Topic | 内容 | 受信者 |
|-----------|------|--------|
| `arn:aws:sns:ap-northeast-1:123456789012:SnapshotReport` | SNAPSHOT REPORT | メール、Slack |

## IAM ポリシー（Lambda 実行ロール）
```json
{
  "Effect":"Allow",
  "Action":[
    "rds:CreateDBSnapshot",
    "rds:DescribeDBSnapshots",
    "sns:Publish",
    "logs:CreateLogGroup",
    "logs:CreateLogStream",
    "logs:PutLogEvents"
  ],
  "Resource":"*"
}
```

## デプロイ手順（CDK なし）
1. Lambda ハンドラを `/opt/lambda/snapshot/` に配置。  
2. IAM ロールを作成し、上記ポリシーを添付。  
3. `aws lambda create-function` で関数を作成。  
4. SNS トピックを作成し、サブスクライバーを追加。
