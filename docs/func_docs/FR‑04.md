# FR‑04 Maintenance Display

## 概要
Agent が RDS（または EC2、Lambda など）など対象サービスの現在の **メンテナンスウィンドウ** を取得し、利用者にわかりやすく提示します。
手動クエリまたはインシデント状況で呼び出します。

## 入力 | 主要パラメータ
| パラメータ | 型 | 説明 |
|-----------|----|------|
| `ServiceName` | string | 対象サービス名 (例: `RDS`) |
| `ResourceARN` | string | 対象リソース ARN |
| `IncludeChanges` | boolean | `true` なら変更予定時刻も取得 |

## 処理フロー
1. Lambda が Service‑Specific API (`DescribeMaintenanceWindow`, `DescribeDBInstances` など) を呼び出し。  
2. 返却された **開始時刻・終了時刻・次回実施予定** を統計・フォーマット化。  
3. 生成したレポートを SNS (`MaintenanceReport`) へ publish。  
4. CloudWatch Log (`/aws/lambda/aiops-maintenance`) にログを残す。

## SNS 通知設定
| SNS Topic | 内容 | 受信者 |
|-----------|------|--------|
| `arn:aws:sns:ap-northeast-1:123456789012:MaintenanceReport` | MAINTENANCE REPORT | メール、Slack |

## IAM ポリシー（Lambda 実行ロール）
```json
{
  "Effect":"Allow",
  "Action":[
    "rds:DescribeDBInstances",
    "ec2:DescribeInstanceStatus",
    "lambda:GetFunctionConfiguration",
    "sns:Publish",
    "logs:CreateLogGroup",
    "logs:CreateLogStream",
    "logs:PutLogEvents"
  ],
  "Resource":"*"
}
```

## デプロイ手順（CDK なし）
1. Lambda ハンドラを `/opt/lambda/maintenance/` に配置。  
2. IAM ロールを作成し、上記ポリシーを添付。  
3. `aws lambda create-function` で関数を作成。  
4. SNS トピックを作成し、サブスクライバーを追加。
