# FR‑05 Slow Query 検出

## 概要
AWS RDS Performance Insights API を利用して、
過去 24 時間における **Slow Query** を検出し、SNS でレポートを配信する機能。

## 入力 | パラメータ
| パラメータ | 型 | 説明 |
|-----------|----|--------|
| `Duration` | integer | 検索期間（秒単位）。デフォルト 86400（24h）。 |
| `MaxResults` | integer | 取得件数の上限。デフォルト 100。 |
| `Filters` | list | `SQLText` への LIKE フィルタ。例：`"SELECT * FROM orders"` |

## 処理フロー
1. EventBridge で毎週日曜 00:00 UTC に Lambda を起動。
2. `GetResourceMetrics` を呼び出し、過去 24 時間の `QueryRuntime` メトリクスを取得。
3. タイムアウト 2000 ms 以上を示すエントリを **Slow Query** と判断。
4. 集計結果を JSON として SNS (`SlowQueryReport`) に publish。
5. Lambda 実行ログを CloudWatch Logs `/aws/lambda/aiops-slow-query-detector` に残す。

## 出力 | レポートJSON
```json
{
  "type": "slowQuery",
  "runAt": "2026-06-10T00:00:00Z",
  "queries": [
    {
      "queryText": "SELECT * FROM orders WHERE created_at < '2025-01-01'",
      "averageRuntimeMs": 15234,
      "executionCount": 34,
      "topDBInstances": ["ap-northeast-1b", "ap-northeast-1c"]
    }
  ]
}
```

## SNS 通知設定
- **トピック**：`arn:aws:sns:ap-northeast-1:123456789012:SlowQueryReport`
- **サブスクライバー**：メール, SQS, Slack webhook などを設定可能。

## デプロイ手順（CDK なし）
1. Lambda ハンドラ（Python/Node.js）を `/opt/lambda/smallquery/` に配置。
2. `create-iam-role.sh` で `PerformanceInsightsRole` を作成（上記 IAM ポリシーを添付）。
3. `aws lambda create-function` で Lambda を作成。
4. `aws events put-rule` でバッチスケジュールを作成。
5. `aws events put-targets` で Lambda をターゲットに設定。
6. SNS トピックを作成し、`aws sns subscribe` で購読を追加。
